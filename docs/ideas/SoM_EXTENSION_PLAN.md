## Phase 1: 透明パネルオーバーレイ（dashboard マーカー強化）

### 概要
キャプチャ時に取得した UI 要素の視覚的な配置を、スクリーンショット画像の上に透明パネルとして重ねる。
現在のマーカー（赤丸＋ツールチップ）を拡張し、要素の形状・サイズ・位置を正確にトレースした透明な矩形パネルとして表示する。

### 目的
- **ホバー時のツールチップ表示精度向上**: 要素の正確な矩形範囲を視覚化
- **クリック/ピンの操作性向上**: 要素の領域が明確になり、小さな要素でも操作しやすい
- **コピー情報の充実**: 矩形情報＋要素階層＋CSSセレクターの完全な情報を提供
- **スクロール対応**: キャプチャ画像が相対的に動いてもパネルが追従

### 実装イメージ
```
スクリーンショット画像（ベース）
  └── 透明パネル群（absolute 配置）
       ├── 要素1: <div.cap-overlay> 枠線+ホバー検出領域
       │   ├── 赤丸バッジ（既存）
       │   └── ツールチップ（既存＋拡張）
       ├── 要素2: <div.cap-overlay> ...
       └── ...
```

### 既存マーカーとの差分
| 項目 | 現在（v3.1.0） | Phase 1 後 |
|------|----------------|-------------|
| 矩形表示 | ホバー時のみ枠線 | 常時薄い枠線＋ホバーで強調 |
| ツールチップ | 上方向に固定 | 画面端で自動的に向き反転 |
| 情報量 | tag, text, coords | + full selector, 階層パス, 属性一覧 |
| コピー | 右クリック1種類 | 右クリック＋ピン状態で内容が変化 |
| 視覚的配置 | マーカーのみ | 要素の形状が透けて見える |

### スクロール対応
`#cap-scroll`（`display: inline-block`）内で画像とマーカー層が一体となってスクロールするため、
キャプチャ画像に対する相対位置がズレない。既に現行実装で対応済み。

### 要素の重複除去
`collectElements()` に `seen` Set による位置ベースの重複除去を追加済み（sw.js 実装）。

## Phase 3: Agent 向けチートシート

### 概要
SoM の情報を agent（LLM）に提供する際、画面画像だけでなく「どの座標に何の要素があるか」を
構造化データとして同時に渡す仕組み。

### メリット
- **要素が存在しない場合**: 「この座標に◯◯ボタンがあるはず」という情報と実際の画面を比較し、
  agent が問題を把握できる
- **要素が存在する場合**: ポインタの座標指示が容易かつ確実になる
- **従来の実装でも不可能ではなかったが、本 Phase で明示的インターフェースとして確立する**

### データ形式（案）
```json
{
  "elements": [
    {
      "id": 1,
      "tag": "button",
      "text": "Submit",
      "selector": "#submit-btn",
      "center": { "x": 850, "y": 1200 },
      "size": { "w": 120, "h": 40 }
    }
  ],
  "viewport": { "width": 1920, "height": 1080 }
}
```

### SoM 画像との併用
- Phase 1 の透明パネルはあくまで dashboard 上の視覚化
- Phase 3 のチートシートは MCP 経由で agent に提供
- SOM 画像のマーキング色は「画面内で未使用の色」または「背景色・ボタン色と被らない色」を動的選択し、
  枠線での強調を行う
