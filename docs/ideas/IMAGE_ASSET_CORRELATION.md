# Captured-image ↔ structure correlation & browsing

**Status:** idea / exploratory (2026-06-04) — not scheduled, low priority. A rough
capture; specifics to be worked out later.

## 概要

ページが読み込む/表示する画像（`<img src>`/`srcset`、CSS background 等のリンク）を、その
**DOM・フロント構造上の位置**と**セッション記録上の時点**に対応づけて記録し、後から「この画像が
UI のどの部分・セッションのどの時点のものか」を辿れるようにする。取得した画像は **個別 / 選択 /
全件**で任意にダウンロードでき、**検索**もできる。

## 想定する部品（粗）

- **記録**: 画像アセットのリンクを、出現した要素・近傍テキスト・構造（component / セレクタ）・
  タイムスタンプと一緒にセッションへ保存。既存アナライザ（text-coords / ui-catalog / dom-snapshot）
  と `source-store` の延長で取れる範囲を使う。
- **索引 / 検索**: URL・alt・近傍テキスト・対応 component などをキーに検索。
- **エクスポート**: 単体 / 選択 / 全件をダウンロード（依存ゼロの `zip-writer` を流用）。
- **対応づけ**: 実行時に観測した画像 ↔ 構造の対応。per-element サムネイル（`som-thumbnails`）と
  相性が良い。

## dev パネル案（おまけ）

いま **viewport 内に入っている画像だけ**を、それぞれ個別のサムネイルとしてリアルタイム表示する
dev パネル。一度に出す枚数の上限は UI 側で調整可能。コスト感: 表示するのが in-view の画像だけ
なら、リアルタイム表示でも静止表示の概ね ~1.x 倍程度の重さに収まりそう（細かく見ると差はあるが）。

## 関連・留意

- 秘匿ガード（[[project_secret_guard]]）の対象領域は画像でもマスク/除外する。
- ダウンロード/エクスポートは外部送信ではなくローカル取得に留める前提。
