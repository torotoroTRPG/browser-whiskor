# E2E テスト検討メモ

## 現状
- 単体テスト・結合テスト: `node:test` + `ws` （依存なし）
- サーバー/メッセージングのテストはカバレッジ十分

## 今後追加する可能性のあるE2Eテスト
- **Playwright** の導入を検討（Seleniumより高速、自動待機あり）
- 実際のブラウザ操作を伴うテストが必要な場合に追加
- 現時点では未確定・検討段階

## 候補シナリオ
- ブラウザ拡張機能のインストール → WebSocket接続 → データ収集
- ダッシュボードの表示・操作
- MCPツール経由のブラウザ操作

## 注意点
- 導入する場合は `npm i -D playwright` のみ追加
- 既存の `node:test` テストには影響しない
- CI/CDにブラウザランタイムのセットアップが必要
